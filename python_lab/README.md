# Лабораторная работа: BST и AVL

В проекте реализованы:
- обычное бинарное дерево поиска (BST),
- самобалансирующееся AVL-дерево,
- скрипт для серии экспериментов по условиям задания,
- автоматическое построение графиков и генерация отчёта.

## Статус Python-прогона (14.04.2026)
- Полный запуск по ТЗ был начат: `10 серий x 20 циклов x 1000 search x 1000 delete`.
- Фактически выполнено `137/200` циклов (до серии 7, цикл 17 включительно).
- Запуск остановлен из-за слишком большого времени на отсортированных данных для обычного BST.

Ключевые наблюдения по времени одного цикла (из live-прогресса):
- series 5, sorted, `n=16384`: около `17s` за цикл;
- series 6, sorted, `n=32768`: около `64s` за цикл;
- series 7, sorted, `n=65536`: около `250-261s` за цикл.

Это ожидаемая деградация BST на отсортированном вводе (вплоть до поведения порядка `O(N^2)` на фазе построения).

В этом репозитории дальнейшее завершение полного эксперимента перенесено в C++-реализацию: см. `../cpp_lab/README.md`.

## Соответствие заданию
- 10 серий тестов: по умолчанию `--series 10`.
- 20 циклов в серии: по умолчанию `--cycles 20`.
- Первые 10 циклов — случайный массив: `--random-cycles 10`.
- Вторые 10 циклов — отсортированный массив.
- Размер массива: `2^(10 + i)`.
- Для каждого цикла:
  - замер вставки всего массива в BST и AVL,
  - 1000 поисков в BST и AVL + среднее время 1 поиска,
  - 1000 поисков в массиве (линейный поиск) + среднее время,
  - 1000 удалений в BST и AVL + среднее время 1 удаления.
- Раздельные графики для random/sorted и отдельная линия поиска по массиву.

## Почему финальный полный прогон переведён в C++
- В Python на старших сериях отсортированных данных время резко растёт из-за деградации BST.
- Даже при корректной реализации суммарное wall-clock время полного прогона становится непрактичным для рабочего цикла.
- C++-версия в `cpp_lab` сохраняет ту же методику и формат выходных артефактов, но рассчитана на завершение полного эксперимента в разумное время.
