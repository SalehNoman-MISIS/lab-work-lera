# Отчёт по лабораторной работе

## Тема
Сравнительный анализ бинарного дерева поиска (BST) и самобалансирующегося AVL-дерева.

## Цель работы
Изучить и реализовать структуры BST и AVL, провести серию вычислительных экспериментов и сравнить производительность операций вставки, поиска и удаления на разных входных данных.

## Постановка задачи
Требуется провести 10 серий экспериментов. В каждой серии выполняется 20 циклов.

- Для первых 10 циклов массив заполняется случайными значениями.
- Для вторых 10 циклов массив заполняется в отсортированном порядке.
- Размер массива для серии i задаётся как:

$$
N_i = 2^{10 + i},\quad i = 0..9
$$

В каждом цикле:
1. Вставить весь массив в BST и AVL, измерить общее время вставки.
2. Выполнить 1000 поисков случайного числа в диапазоне значений массива для BST и AVL; измерить общее время и вычислить время одной операции поиска.
3. Выполнить 1000 поисков по массиву (линейный поиск); измерить общее время и время одной операции.
4. Выполнить 1000 операций удаления в BST и AVL; измерить общее время и время одной операции.

## Реализация
- Язык программирования: Python 3.
- Реализация структур данных:
  - BST без балансировки;
  - AVL с поддержкой высоты узлов и ротаций (LL, RR, LR, RL).
- Для измерений используется `time.perf_counter()`.
- Для построения графиков используется `matplotlib`.

Основные файлы проекта:
- `src/trees.py` — структуры BST и AVL.
- `src/benchmark.py` — экспериментальный стенд.

## Методика эксперимента
1. Для каждой серии формируется размер массива $N_i$.
2. Для каждого цикла генерируется набор:
   - `random`: уникальные случайные значения,
   - `sorted`: последовательность 0..N-1.
3. На одном и том же наборе сравниваются BST и AVL.
4. Результаты каждого цикла сохраняются в таблицу сырых данных.
5. По каждой серии и типу набора вычисляются средние значения и стандартные отклонения.

## Полученные артефакты
После запуска эксперимента в проекте сгенерированы:
- `demo_outputs/raw_results.csv` — все замеры по циклам;
- `demo_outputs/aggregated_results.csv` — усреднённые метрики;
- `demo_outputs/plots/random_dataset.png` — графики для случайного набора;
- `demo_outputs/plots/sorted_dataset.png` — графики для отсортированного набора;

Параметры фактического прогона для валидации и анализа в этом отчёте:
- число серий: 2;
- число циклов в серии: 2 (1 random + 1 sorted);
- число поисков: 100;
- число удалений: 100.

Для полной сдачи по ТЗ требуется запуск с параметрами 10 серий × 20 циклов × 1000 операций.

## Анализ результатов
1. Случайные данные (random).
   - На размерах 1024 и 2048 вставка в BST оказалась быстрее AVL, так как AVL тратит дополнительное время на поддержание баланса (ротации и пересчёт высот).
   - Поиск в AVL стабильно быстрее BST: для n=2048 среднее время 1 поиска составило 1.585e-6 c для AVL против 2.708e-6 c для BST.
   - По удалению в random-наборе BST в данном запуске быстрее AVL (для n=2048: 3.165e-6 c против 1.872e-5 c), что объясняется накладными расходами балансировки в AVL.

2. Отсортированные данные (sorted).
   - Для вставки наблюдается сильная деградация BST: при n=2048 время вставки массива в BST составило 0.370 c, а в AVL — 0.039 c (примерно в 9.46 раза быстрее).
   - Для поиска деградация ещё более заметна: при n=2048 BST дал 1.42459e-4 c на 1 операцию, AVL — 1.669e-6 c (примерно в 85 раз быстрее AVL).
   - Удаление на sorted-наборе также значительно лучше у AVL: 1.724e-5 c против 1.48032e-4 c у BST (примерно в 8.6 раза).

3. Сравнение поиска в деревьях и массиве.
   - Линейный поиск в массиве растёт пропорционально размеру и заметно медленнее поиска в AVL.
   - Для n=2048 в random-наборе: массив 5.9984e-5 c/операцию, AVL 1.585e-6 c/операцию (около 38 раз быстрее AVL).
   - Для n=2048 в sorted-наборе: массив 3.1769e-5 c/операцию, AVL 1.669e-6 c/операцию (около 19 раз быстрее AVL).

4. Теоретическое объяснение результатов.
   - BST: средний случай близок к $O(\log N)$, но в худшем случае (например, отсортированный ввод) высота становится порядка $N$, и операции стремятся к $O(N)$.
   - AVL: за счёт балансировки поддерживает высоту порядка $\log N$, поэтому вставка/поиск/удаление остаются $O(\log N)$ даже на «плохом» входе.
   - Линейный поиск в массиве всегда $O(N)$.

## Вывод
По результатам эксперимента подтверждены ожидаемые свойства структур данных.

1. Обычный BST демонстрирует сильную деградацию на отсортированном наборе данных: при росте размера массива время вставки, поиска и удаления увеличивается значительно быстрее, чем у AVL.
2. AVL-дерево обеспечивает более стабильное поведение на всех типах входных данных благодаря самобалансировке, особенно заметно в операциях поиска и удаления на отсортированных наборах.
3. Поиск в массиве (линейный) существенно уступает поиску в деревьях на данных размерах, а AVL показывает наилучшее сочетание устойчивости и скорости.

Следовательно, для практических задач с неопределённым или потенциально неблагоприятным распределением входных данных AVL-дерево является более надёжным выбором по времени выполнения операций.

## Практическое замечание по полному прогону
Полный запуск с параметрами ТЗ в Python для обычного BST на отсортированных данных имеет крайне высокую вычислительную стоимость из-за деградации до $O(N^2)$ при вставке. На старших сериях это приводит к очень длительному времени эксперимента.

Поэтому для сдачи обычно используют один из двух форматов:
1. Запуск полного эксперимента на более производительной среде и включение финальных файлов `outputs/*` в отчёт.
2. Запуск на уменьшенных параметрах с явным указанием ограничений и обязательным теоретическим обоснованием асимптотики, что и сделано в данном документе.
